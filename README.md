## What Is Proof of Alpha?

**Proof of Alpha (PoA)** is a tokenomics primitive in which a token's deflationary events are mathematically bound to verified, externally-audited trading performance — not to protocol usage, not to staking, and not to self-reporting.

```
Proof of Work   →  burn energy      to emit tokens
Proof of Stake  →  lock capital     to validate
Proof of Alpha  →  generate alpha   to burn tokens
```

Every burn event in a PoA system contains:
- The USD value of the verified trading profit that triggered it
- A cryptographic hash of the original trade record from the auditing institution
- An on-chain event linking both, permanently and immutably

**No burn without proof. No proof without external verification.**

## Why It Has Not Existed Before

Building a PoA system requires four simultaneous capabilities:

1. A real trading system operating in production on live capital
2. Access to institutional FIX Protocol infrastructure (broker-level)
3. Deep knowledge of FIX — the standard messaging protocol of institutional finance since 1992
4. The engineering to cryptographically bridge FIX execution reports to a blockchain

These four requirements have never coexisted in a single project before ZNT.

## The Core Invariants

Any compliant PoA implementation must uphold these invariants:

**1. No self-reporting**
The entity generating trading performance cannot be the sole source of burn triggers. An independent, regulated third party must verify the performance.

**2. Immutable audit trail**
Every burn event must contain a cryptographic reference to the original trade record. The on-chain event and the off-chain record must verify each other.

**3. No mint**
A PoA token cannot have a public mint function. Supply is fixed at deploy. The only direction is down.

**4. Anti-replay**
Each trade record can trigger at most one burn event. Replay protection must be enforced at the contract level.

**5. External verification incentive alignment**
The auditing institution (prop firm, exchange, regulated broker) must be economically incentivized to detect and reject fraud. Their business model must depend on accurate performance verification.

## Comparing Approaches

| Property | Protocol Revenue | Yield Token | **Proof of Alpha** |
|---|---|---|---|
| Backing source | Protocol usage | Inflation | **Verified trading P&L** |
| External verification | None | None | **Regulated institution** |
| Supply direction | Variable | Inflationary | **Only downward** |
| Backing grows over time | Depends on usage | No | **Yes — compounding** |
| Manipulation resistance | Medium | Low | **Institutional-grade** |
| Self-reporting risk | Yes | Yes | **No — external auditor** |
